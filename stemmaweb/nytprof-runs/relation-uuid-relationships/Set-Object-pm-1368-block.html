54=head1 DESCRIPTION
55
56This modules implements a set of objects, that is, an unordered
57collection of objects without duplication.
58
59The term I<objects> is applied loosely - for the sake of
60L<Set::Object>, anything that is a reference is considered an object.
61
62L<Set::Object> 1.09 and later includes support for inserting scalars
63(including the empty string, but excluding C<undef>) as well as
64objects. This can be thought of as (and is currently implemented as)
65a degenerate hash that only has keys and no values. Unlike objects
66placed into a Set::Object, scalars that are inserted will be flattened
67into strings, so will lose any magic (eg, tie) or other special bits
68that they went in with; only strings come out.
69

152=head2 weaken
153
154Makes all the references in the set "weak" - that is, they do not
155increase the reference count of the object they point to, just like
156L<Scalar::Util|Scalar::Util>'s C<weaken> function.
157
158This was introduced with Set::Object 1.16, and uses a brand new type

160C<weaken>, please reduce your problem to a test script before
161submission.

166L<Set::Object::Weak> for more.

300=head1 FUNCTIONS
301
302The following functions are defined by the Set::Object XS code for
303convenience; they are largely identical to the versions in the
304Scalar::Util module, but there are a couple that provide functions not
305catered to by that module.
306
307Please use the versions in L<Scalar::Util> in preference to these
308functions.
309
310=over
311
312=item B<blessed>
313
314Returns a true value if the passed reference (RV) is blessed. See
315also L<Acme::Holy>.
316
317=item B<reftype>
318
319A bit like the perl built-in C<ref> function, but returns the I<type>
320of reference; ie, if the reference is blessed then it returns what
321C<ref> would have if it were not blessed. Useful for "seeing through"
322blessed references.
323
324=item B<refaddr>
325
326Returns the memory address of a scalar. B<Warning>: this is I<not>
327guaranteed to be unique for scalars created in a program; memory might
328get re-used!
329
330=item B<is_int>, B<is_string>, B<is_double>
331
332A quick way of checking the three bits on scalars - IOK (is_int), NOK
333(is_double) and POK (is_string). Note that the exact behaviour of
334when these bits get set is not defined by the perl API.
335
336This function returns the "p" versions of the macro (SvIOKp, etc); use
337with caution.
338
339=item B<is_overloaded>
340
341A quick way to check if an object has overload magic on it.
342
343=item B<ish_int>
344
345This function returns true, if the value it is passed looks like it
346I<already is> a representation of an I<integer>. This is so that you
347can decide whether the value passed is a hash key or an array
348index.
349
350=item B<is_key>
351
352This function returns true, if the value it is passed looks more like
353an I<index> to a collection than a I<value> of a collection.
354
355But wait, you say - Set::Object has no indices, one of the fundamental
356properties of a Set is that it is an I<unordered collection>. Which
357means I<no indices>. Well, if this module were ever to be derived to
358be a more general multi-purpose collection, then this (and C<ish_int>)
359might be a good function to use to distinguish different types of
360indexes from values.
361
362=item B<get_magic>
363
364Pass to a scalar, and get the magick wand (C<mg_obj>) used by the weak
365set implementation. The return will be a list of integers which are
366pointers to the actual C<ISET> structure. Whatever you do don't
367change the array :). This is used only by the test suite, and if you
368find it useful for something then you should probably conjure up a
369test suite and send it to me, otherwise it could get pulled.
370
371=back

401=head1 SERIALIZATION
402
403It is possible to serialize C<Set::Object> objects via L<Storable> and
404duplicate via C<dclone>; such support was added in release 1.04. As
405of C<Set::Object> version 1.15, it is possible to freeze scalar items,
406too.
407
408However, the support for freezing scalar items introduced a backwards
409incompatibility. Earlier versions than 1.15 will C<thaw> sets frozen
410using Set::Object 1.15 and later as a set with one item - an array
411that contains the actual members.
412
413Additionally, version 1.15 had a bug that meant that it would not
414detect C<freeze> protocol upgrades, instead reverting to pre-1.15
415behaviour.
416
417C<Set::Object> 1.16 and above are capable of dealing correctly with
418all serialized forms, as well as correctly aborting if a "newer"
419C<freeze> protocol is detected during C<thaw>.
