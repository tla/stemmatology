94=head1 NAME
95
96Data::Stream::Bulk - N at a time iteration API
97
98=head1 VERSION
99
100version 0.11
101
102=head1 SYNOPSIS
103
104 # get a bulk stream from somewere
105 my $s = Data::Stream::Bulk::Foo->new( ... );
106
107 # can be used like this:
108 until ( $s->is_done ) {
109 foreach my $item ( $s->items ) {
110 process($item);
111 }
112 }
113
114 # or like this:
115 while( my $block = $s->next ) {
116 foreach my $item ( @$block ) {
117 process($item);
118 }
119 }
120
121=head1 DESCRIPTION
122
123This module tries to find middle ground between one at a time and all at once
124processing of data sets.
125
126The purpose of this module is to avoid the overhead of implementing an
127iterative api when this isn't necessary, without breaking forward
128compatibility in case that becomes necessary later on.
129
130The API optimizes for when a data set typically fits in memory and is returned
131as an array, but the consumer cannot assume that the data set is bounded.
132
133The API is destructive in order to minimize the chance that resultsets are
134leaked due to improper usage.
135
136=head1 API
137
138=head2 Required Methods
139
140The API requires two methods to be implemented:
141
142=over 4
143
144=item is_done
145
146Should return true if the stream is exhausted.
147
148As long as this method returns a false value (not done) C<next> could
149potentially return another block.
150
151=item next
152
153Returns the next block.
154
155Note that C<next> is not guaranteed to return an array reference, even if
156C<is_done> returned false prior to calling it.
157
158=back
159
160=head2 Convenience Methods
161
162=over 4
163
164=item items
165
166This method calls C<next> and dereferences the result if there are pending
167items.
168
169=item all
170
171Force evaluation of the entire resultset.
172
173Note that for large data sets this might cause swap thrashing of various other
174undesired effects. Use with caution.
175
176=item cat @streams
177
178Concatenates this stream with @streams, returning a single stream.
179
180=item list_cat @tail
181
182Returns a possibly cleaned up list of streams.
183
184Used by C<cat>.
185
186Overridden by L<Data::Stream::Bulk::Array>, L<Data::Stream::Bulk::Cat> and
187L<Data::Stream::Bulk::Nil> to implement some simple short circuiting.
188
189=item filter $filter
190
191Applies a per-block block filter to the stream.
192
193Returns a possibly new stream with the filtering layered.
194
195C<$filter> is invoked once per block and should return an array reference to
196the filtered block.
197
198=item chunk $chunk_size
199
200Chunks the input stream so that each block returned by C<next> will have at
201least C<$chunk_size> items.
202
203=item loaded
204
205Should be overridden to return true if all the items are already realized (e.g.
206in the case of L<Data::Stream::Bulk::Array>).
207
208Returns false by default.
209
210When true calling C<all> is supposed to be safe (memory usage should be in the
211same order of magnitude as stream's own usage).
212
213This is typically useful when tranforming an array is easier than transorming a
214stream (e.g. optional duplicate filtering).
215
216=back
