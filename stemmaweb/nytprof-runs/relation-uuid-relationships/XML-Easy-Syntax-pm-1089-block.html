56=head1 REGULAR EXPRESSIONS
57
58Each of these regular expressions corresponds precisely to one of
59the productions in the EBNF grammar in the XML 1.0 specification.
60Well-formedness constraints that are not expressed in the EBNF are
61I<not> checked by the regular expressions; these are noted in the
62documentation below. The regular expressions do not include any anchors,
63so to check whether an entire string matches a production you must supply
64the anchors yourself.
65
66=head2 Syntax pieces
67
68=over
69
70=item $xml10_char_rx
71
72Any single character that is acceptable to XML 1.0. This includes most
73Unicode characters (up to codepoint 0x10ffff). The excluded codepoints
74are the sentinels 0xfffe and 0xffff, the surrogate blocks, and most of
75the C0 control characters (0x00 to 0x1f, except for 0x09 (tab), 0x0a
76(linefeed/newline), and 0x0d (carriage return)).
77
78It is a rule of XML that all characters making up an XML document
79must be in this permitted set. The grammar productions can only match
80sequences of acceptable characters. This rule is enforced by the regular
81expressions in this module.
82
83Furthermore, it is a rule that the character data in a document cannot
84even I<represent> a character outside the permitted set. This is
85expressed as a well-formedness constraint on character references.
86
87=cut

1128=head1 BUGS
1129
1130Many of these regular expressions are liable to tickle a serious bug in
1131perl's regexp engine. The bug is that the C<*> and C<+> repeat operators
1132don't always match an unlimited number of repeats: in some cases they are
1133limited to 32767 iterations. Whether this bogus limit applies depends
1134on the complexity of the expression being repeated, whether the string
1135being examined is internally encoded in UTF-8, and the version of perl.
1136In some cases, but not all, a false match failure is preceded by a warning
1137"Complex regular subexpression recursion limit (32766) exceeded".
1138
1139This bug is present, in various forms, in all perl versions up to at
1140least 5.8.9 and 5.10.0. Pre-5.10 perls may also overflow their stack
1141space, in similar circumstances, if a resource limit is imposed.
1142
1143There is no known feasible workaround for this perl bug. The regular
1144expressions supplied by this module will therefore, unavoidably, fail
1145to accept some lengthy valid inputs. Where this occurs, though, it is
1146likely that other regular expressions being applied to the same or related
1147input will also suffer the same problem. It is pervasive. Do not rely
1148on this module (or perl) to process long inputs on affected perl versions.
1149
1150This bug does not affect the L<XML::Easy::Text> parser.
