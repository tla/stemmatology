385=head1 NAME
386
387Graph::Reader::Dot - class for reading a Graph instance from Dot format
388
389=head1 SYNOPSIS
390
391 use Graph::Reader::Dot;
392 use Graph;
393
394 $reader = Graph::Reader::Dot->new();
395 $graph = $reader->read_graph('mygraph.dot');
396
397=head1 DESCRIPTION
398
399B<Graph::Reader::Dot> is a class for reading in a directed graph
400in the file format used by the I<dot> tool (part of the AT+T graphviz
401package).
402
403B<Graph::Reader::Dot> is a subclass of B<Graph::Reader>,
404which defines the generic interface for Graph reader classes.
405
406=head1 METHODS AND CONFIGURATION
407
408=head2 C<new()>
409
410Constructor - generate a new reader instance.
411
412 $reader = Graph::Reader::Dot->new();
413
414This doesn't take any arguments.
415
416=head2 C<read_graph()>
417
418Read a graph from a file:
419
420 $graph = $reader->read_graph( $file );
421
422The C<$file> argument can be either a filename
423or a filehandle of a previously opened file.
424
425=head2 C<$Graph::Reader::Dot::UseNodeAttr>
426
427Controls, if implicit node attributes given by the dot directive C<node[]> will be merged into (new) nodes.
428Setting it to C<0> or C<undef> (default) will not disable this feature.
429Setting it to any other value will enable this feature.
430
431=head2 C<$Graph::Reader::Dot::UseEdgeAttr>
432
433Controls, if implicit edge attributes given by the dot directive C<edge[]> will be merged into edges.
434Setting it to C<0> or C<undef> (default) will not disable this feature.
435Setting it to any other value will enable this feature.
436
437=head1 RESTRICTIONS
438
439=over 4
440
441=item *
442
443Default (graph) attributes in subgraphs (i.e. inside C<{}>) are not processed.
444
445=item *
446
447Sub nodes as used by dot's C<record> node shape are supported.
448
449=item *
450
451Undirected graphs will be treated as directed graphs.
452This means that the C<--> edge operator works as the C<-E<gt>> edge operator.
453
454=item *
455
456Be aware that you are loosing scope information on writing back the graph.
457
458=item *
459
460Multiple C<node[]> or C<edge[]> statements in the same scope are not correctly supported.
461
462=back
