305
306 use Data::Visitor::Callback;
307
308 my $v = Data::Visitor::Callback->new(
309 # you can provide callbacks
310 # $_ will contain the visited value
311
312 value => sub { ... },

335=head1 DESCRIPTION
336
337This is a L<Data::Visitor> subclass that lets you invoke callbacks instead of
338needing to subclass yourself.
339
340=encoding utf8
341
342=head1 METHODS
343
344=over 4
345
346=item new %opts, %callbacks
347
348Construct a new visitor.
349
350The options supported are:
351
352=over 4
353
354=item ignore_return_values
355
356When this is true (off by default) the return values from the callbacks are
357ignored, thus disabling the fmapping behavior as documented in
358L<Data::Visitor>.
359
360This is useful when you want to modify $_ directly
361
362=item tied_as_objects
363
364Whether ot not to visit the L<perlfunc/tied> of a tied structure instead of
365pretending the structure is just a normal one.
366
367See L<Data::Visitor/visit_tied>.
368
369=back
370
371=back
372
373=head1 CALLBACKS
374
375Use these keys for the corresponding callbacks.
376
377The callback is in the form:
378
379 sub {
380 my ( $visitor, $data ) = @_;
381
382 # or you can use $_, it's aliased
383
384 return $data; # or modified data
385 }
386
387Within the callback $_ is aliased to the data, and this is also passed in the
388parameter list.
389
390Any method can also be used as a callback:
391
392 object => "visit_ref", # visit objects anyway
393
394=over 4
395
396=item visit
397
398Called for all values
399
400=item value
401
402Called for non objects, non container (hash, array, glob or scalar ref) values.
403
404=item ref_value
405
406Called after C<value>, for references to regexes, globs and code.
407
408=item plain_value
409
410Called after C<value> for non references.
411
412=item object
413
414Called for blessed objects.
415
416Since L<Data::Visitor/visit_object> will not recurse downwards unless you
417delegate to C<visit_ref>, you can specify C<visit_ref> as the callback for
418C<object> in order to enter objects.
419
420It is reccomended that you specify the classes (or base classes) you want
421though, instead of just visiting any object forcefully.
422
423=item Some::Class
424
425You can use any class name as a callback. This is colled only after the
426C<object> callback.
427
428If the object C<isa> the class then the callback will fire.
429
430These callbacks are called from least derived to most derived by comparing the
431classes' C<isa> at construction time.
432
433=item object_no_class
434
435Called for every object that did not have a class callback.
436
437=item object_final
438
439The last callback called for objects, useful if you want to post process the
440output of any class callbacks.
441
442=item array
443
444Called for array references.
445
446=item hash
447
448Called for hash references.
449
450=item glob
451
452Called for glob references.
453
454=item scalar
455
456Called for scalar references.
457
458=item tied
459
460Called on the return value of C<tied> for all tied containers. Also passes in
461the variable as the second argument.
462
463=item seen
464
465Called for a reference value encountered a second time.
466
467Passes in the result mapping as the second argument.
468
469=back
