27=head1 DESCRIPTION
28
29TEI XML is a wonderful thing. The elements defined therein allow a
30transcriber to record and represent just about any feature of a text that
31he or she encounters.
32
33The problem is the transcription itself. When I am transcribing a
34manuscript, especially if that manuscript is in a bunch of funny characters
35on the keymap for another language, I do not want to be switching back and
36forth between keyboard layouts in order to type "<tag attr="attr>"
37arrow-arrow-arrow-arrow-arrow "</tag> every six seconds. It's prone to
38typo, it's astonishingly slow, and it makes my wrists hurt just to think
39about it. I also don't really want to fire up an XML editor, select the
40words or characters that need to be tagged, and click a lot. That way is
41not prone to typo, but it's still pretty darn slow, and it makes my wrists
42hurt B<even more> to think about.
43
44Text::TEI::Markup is my solution to that problem. It defines a bunch of
45single- or double-character sigils that represent tags. These are a lot
46faster and easier to type; I don't have to worry about typos; and I can do
47it all with a plain text editor, thus minimizing use of the mouse.
48
49I have tried to pick sigils that don't conflict with characters that are
50found in manuscripts. I have succeeded for my particular set of
51manuscripts, but I have not succeeded for the general case. If you like the
52idea behind this module, you are still almost guaranteed to hate the sigils
53I've picked. That's okay; you can re-define them.
54
55=head2 Extra bonus solution: word wrapping with <w/> and <seg/>
56
57Even if you are happy as a clam in the graphical XML editor of your choice,
58this module exports a function that may be useful to you. The TEI P5
59guidelines include a module called "analysis", which allows the user to tag
60sentences, clauses, words, morphemes, or any other sort of semantic segment
61of a text. This is really good for programmatic applications, but very
62boring and repetitive to have to tag.
63
64The function B<word_tag_wrap> solves part of this problem for you. It takes
65an XML string as input, looks for words (defined by whitespace separation)
66and returns an XML string with each of these words wrapped in an
67appropriate tag. If the word has complex elements (e.g. editorial
68expansion), it will be wrapped in a <seg type="word/> tag. If not, it will
69be in a simple <w/> tag. It handles line breaks and page breaks within
70words, as long as there is no trailing whitespace before the <lb/> (or
71<pb/>) tag, and as long as the whitespace after the tag contains a carriage
72return.
73
74=head1 MARKUP SYNTAX
75
76The input file has a header and a body. The header begins with a '=HEAD'
77tag, and consists of a colon-separated list of key_value pairs. These keys,
78which are case insensitive, get directly substituted into an XML template;
79the idea is that your TEI header won't change very much between files, so
80you write it once with template values, pass it to &to_xml, and the
81substitution happens as if by magic. The keyword /MAIN/i is reserved for
82the content between the <body></body> tags - that is, all the content that
83will be generated after the '=BODY' tag.
84
85A very simple template looks like this:
86 <?xml version="1.0" encoding="UTF-8">
87 <TEI>
88 <teiHeader>
89 <fileDesc>
90 <titleStmt>
91 <title>__TITLE__</title>
92 <author__AUTHOR__</author>
93 <respStmt xml:id="#__MYINITIALS__">
94 <resp>Transcription by</resp>
95 <name>__MYNAME__</name>
96 </respStmt>
97 </titleStmt>
98 </fileDesc>
99 </teiHeader>
100 <text>
101 <body>
102 __MAIN__
103 </body>
104 </text>
105 </TEI>
106
107Your input file should then begin something like this:
108
109 =HEAD
110 title:My Summer Vacation: a novel
111 author:John Smith
112 myinitials:tla
113 myname:Tara L Andrews
114 =BODY
115 The ^real^ text b\e\gins +(above)t+here.
116 ...
117
118
119The real work begins after the '=BODY' tag. The currently-defined sigil
120list is:
121
122 %SIGILS = (
123 'comment' => '##',
124 'add' => '+',
125 'del' => '-',
126 'subst' => "\x{b1}", # Unicode PLUS-MINUS SIGN
127 'div' => "\x{a7}", # Unicode SECTION SIGN
128 'p' => "\x{b6}", # Unicode PILCROW SIGN
129 'ex' => '\\',
130 'expan' => '^',
131 'supplied' => '@',
132 'abbr' => [ '{', '}' ],
133 'num' => '%',
134 'pb' => [ '[', ']' ],
135 'cb' => '|',
136 'hi' => '*',
137 'unclear' => '?',
138 'q' => "\x{2020}", # Unicode DAGGER
139 );
140
141Non-identical matched sets of sigla (e.g. '{}' for abbreviations) should be
142specified in a listref, as seen here.
143
144The "add" and "del" sigils have an extra convenience feature - anything
145that appears in parentheses immediately after the add/del opening sigil ( +
146or - in the examples above) will get added as an attribute. If the string
147in parentheses has no '=' sign in it, the attribute for the "add" tag will
148be "place", and the attribute for the "del" tag will be "type". Ergo:
149
150 +(margin)This is an addition+-(overwrite)and a deletion- to the sentence.
151
152will get translated to
153
154 <add place="margin">This is an addition</add>
155 <del type="overwrite">and a deletion</del> to the sentence.
156
157This behavior ought to be more configurable and/or flexible; make it worth
158my while.
159
160Whitespace is only significant at the end of lines. If a line which
161contains non-tag text (i.e. words) ends in whitespace, it is assumed that
162the previous word is a complete word. If the line ends with a
163non-whitespace character, it is assume that the word continues onto the
164next line.
165
166All the sigils must be balanced, and they must nest properly. Remember that
167this is a shorthand for XML. I could be convinced to try to autocorrect
168some unbalanced sigils, but it would be worth at least a few pints of cider
169(or, of course, a patch.)

473=item B<word_tag_wrap>( $xml_string )
474
475Takes a string containing a TEI XML document, and returns that
476document with all its words wrapped in <w/> (or <seg/>) tags. A
477"word" is defined as a series of text characters separated by
478whitespace. A word can have a line break, or even a page break, in
479the middle; if this is the case, there I<may not> be any whitespace
480between the end of the first word segment and the <lb/> (or <pb/>)
481tag. Conversely, there I<must> be whitespace separating the <lb/> (or
482<pb/>) from a complete word.
483
484=cut

525sub _wrap_children {

537 # Get the text.

540 # Carriage returns should only occur after <lb/> elements,
541 # and the spaces around them should therefore be insignificant.

544 # pretend that the node isn't there at all.

555 # word should just be closed. If there are no words
556 # at all, it was just a space. If the first word was
557 # all there is, we haven't encountered a space yet and
558 # need to keep the word open.
559 if( @words ) {
560 my $first = shift @words;
561 $open_word_node->appendText( $first ) if $first;
562 } else {
563 $open_word_node = undef unless @words;
564 }
565 }
566
567 foreach( @words ) {
568 # Skip whitespace "words"
569 next unless /\S/;
570
571 # Make a new node for the word
572 my $word_node = XML::LibXML::Element->new( 'w' );
573 $word_node->setNamespace( $docns );
574 $word_node->appendText( $_ );
575 $new_node->appendChild( $word_node );
576 # ...and keep it open until we find a new word or a space
577 $open_word_node = $word_node;
578 }
579
580 # Close the last word node if our text node ends in a space.
581 if( $str =~ /\s+$/s ) {
582 $open_word_node = undef;
583 }
584 } else {
585 my $wrapped_child;
586 if ( ref( $c ) ne 'XML::LibXML::Comment' && $c->textContent ne ''
587 && $c->textContent =~ /\s+/ ) {
588 # Recurse on any node that itself contains whitespace-separated text.
589 my $new_c = _wrap_children( $c );
590 $wrapped_child = ( $c->toString() ne $new_c->toString() );
591 $c = $new_c;
592 }
593
594 # If there is an open word node, make it a seg and append
595 # our result there; if the child has text content but no
596 # word children, wrap it in a new seg; otherwise just pass
597 # it on through.
598 if( $open_word_node ) {
599 $open_word_node->setNodeName( 'seg' );
600 $open_word_node->setAttribute( 'type', 'word' );
601 $open_word_node->appendChild( $c );
602 } elsif( ref( $c ) eq 'XML::LibXML::Comment' || $c->textContent eq ''
603 || $wrapped_child ) {
604 $new_node->appendChild( $c );
605 } else {
606 my $segment_node = XML::LibXML::Element->new( 'seg' );
607 $segment_node->setNamespace( $docns );
608 $segment_node->setAttribute( 'type', 'word' );
609 $segment_node->appendChild( $c );
610 $new_node->appendChild( $segment_node );
611 # Keep it open in case there is not a leading space on the next
612 # text node.
613 $open_word_node = $segment_node;
614 }
615 }
616 }
617
618 return $new_node;
619}
